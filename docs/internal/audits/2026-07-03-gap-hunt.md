**Status legend:** `FIXED` = failing test + fix landed on this branch · `RESOLVED`
= closed indirectly by another fix · `CATALOGUED` = confirmed/reproduced, fix
recommended but not applied (blast radius, product call, or hardware verification
needed) · `SUSPECTED` = strong lead, not fully substantiated · `COVERAGE` = no bug,
a missing-test gap.

PR #23's room core is largely sound — the room-resolution helpers
(`channelRoomId` / `nodeRoomIds` / `resolveSelectionRoom`), the per-level scope-target
expansion (`addNodeScopeTargets` union vs `addNodeResourceTargets` single-room), the
meter per-channel filter (`filterMeterFrameForUser`), and the ad-hoc recording and
schedule **create** paths all correctly enforce the shared-node isolation invariant.
The confirmed gaps clustered where a request or a later event **changes the room a
resource resolves to** — schedule update, channel reassignment, scheduled-run time —
and one authorization skip on the room-less edge.
